📖 What is React?

React is a JavaScript library for building user interfaces

🔷 Developed by Facebook (Meta) in 2013

🧩 Component-Based Architecture: Build encapsulated components that manage their own state

📝 Declarative: Design interactive UIs for each state, React updates efficiently

⚡ Virtual DOM: Optimizes rendering performance

📱 React Native: Extends to mobile development

💡 Key Concept: React is a library, not a framework - it focuses only on the view layer

✨ Benefits of Using React

BenefitDescription
🔄 ReusabilityComponents can be reused across the application
⚡ PerformanceVirtual DOM minimizes direct DOM manipulation
🔍 SEO FriendlyCan render on the server (Next.js)
🌍 Strong EcosystemRich libraries, tools, and community support

🔑 Key Files:
• index.html: Contains <div id="root"></div> where React mounts
• main.jsx: Entry point that renders App component
• App.jsx: Your main application component

🎯 Understanding main.jsx

File Location: src/main.jsx

import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'
import './index.css'

// React 18+ uses createRoot
ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
)
📝 Explanation:
• ReactDOM.createRoot(): Creates a React root for the DOM element
• document.getElementById('root'): Targets the root div in index.html
• React.StrictMode: Development helper for highlighting potential problems
• <App />: Renders the main App component

💻 Creating Your First React Component

File Location: src/App.jsx

import React from 'react'
import './App.css'

function App() {
  // JavaScript logic goes here
  const greeting = "Welcome to React!"
  const studentCount = 25
  
  // JSX - JavaScript XML
  return (
    <div className="App">
      <h1>Hello, Advanced Front-End Developers! 🚀</h1>
      <p>{greeting}</p>
      <p>Number of students: {studentCount}</p>
    </div>
  )
}

export default App
📌 JSX Rules:
• Must have a single parent element
• Use className instead of class
• JavaScript expressions inside {}
• Self-closing tags like <img />

🔍 JSX Deep Dive

JSX is syntactic sugar for React.createElement()

This JSX:

<h1 className="title">Hello World</h1>

Transforms to:

React.createElement('h1', { className: 'title' }, 'Hello World')

Complex Example:

<div>
  <h1>Welcome</h1>
  <p>Total: {5 + 10}</p>
</div>
💡 Why JSX? Readability, familiarity, and better tooling support

⚠️ Common Mistakes & Best Practices

❌ Common Mistakes

// Missing return
const MyComponent = () => {
  <h1>Hello</h1>
}

// Wrong: 'class' not 'className'
<div class="container"></div>

// Missing key prop
{items.map(item => <li>{item}</li>)}

✅ Best Practices

// Use explicit return
const MyComponent = () => {
  return <h1>Hello</h1>
}

// Use className
<div className="container"></div>

// Always use unique key
{items.map(item => (
  <li key={item.id}>{item}</li>
))}

import React from 'react'
import './Course.css'

function Course({ title, description, duration, level, onEnroll }) {

      </div>

        Enroll Now
      </button>
    </div>
  )
}
